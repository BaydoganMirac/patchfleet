# Security policy

## Supported versions

Patchfleet has not released a supported version yet. Security fixes currently
target the latest code on the default branch.

## Report a vulnerability

Report vulnerabilities privately through
[GitHub Security Advisories](https://github.com/BaydoganMirac/patchfleet/security/advisories/new).

Include the affected component, reproduction steps, impact, and the smallest
safe proof of concept you can provide. Do not open a public issue for an
unfixed vulnerability.

Never include credentials, access tokens, private keys, customer source code,
prompts, transcripts, terminal output, or raw customer data in a report. Use
synthetic or redacted examples. If sensitive data was exposed, describe its
type and scope without copying it into GitHub.

Please allow time to investigate and coordinate a fix before public disclosure.
